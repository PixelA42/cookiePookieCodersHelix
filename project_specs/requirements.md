
# HeatREco — Requirements (AI-Driven Version)

## Project Scope

HeatREco is a simple web platform that connects industrial heat producers (factories, data centers, refineries) with nearby heat consumers (greenhouses, cold-storage units, district heating networks). The goal is to treat waste heat as a transferable utility, enabling facilities with surplus heat to match with those who need it, reducing waste and energy costs.

**Frontend:** Next.js (React)
**Backend:** Python (FastAPI)
**Database:** Cloud-hosted (must be online)

---

## Requirements

### 1. User Registration & Authentication
- Users can register and log in as either a Producer or Consumer.
- Registration requires organization name, email, password, and role selection.
- Secure authentication (JWT or similar) is required for all protected endpoints.

### 2. Facility Profile Submission
- Producers submit a profile describing their surplus heat (location, temperature, output, schedule, etc.).
- Consumers submit a profile describing their thermal demand (location, temperature needs, flow rate, schedule, etc.).
- Profiles are stored in the online database.

### 3. AI-Powered Matchmaking
- The backend uses an AI model to recommend and rank the best Producer–Consumer matches based on all available profile data (location, temperature, schedule, etc.).
- The AI model learns from historical match outcomes and user feedback to improve recommendations over time.
- The system provides a compatibility score (0–100) for each match, generated by the AI model.

### 4. Match Dashboard
- Logged-in users see a dashboard listing their top matches, ranked by compatibility score.
- Each match shows key info: counterpart facility name, score, and main profile details.
- A map view displays the user's facility and matched facilities.

### 5. Feedback Loop
- Users can provide feedback on matches (e.g., "useful", "not useful", or similar).
- The AI model uses this feedback to retrain and improve future recommendations.

### 6. API
- All frontend/backend communication is via a versioned REST API (`/api/v1/`).
- All data operations (registration, profile CRUD, matchmaking, feedback) are exposed as API endpoints.

### 7. Online Database
- All data is stored in a cloud-hosted, online database (e.g., PostgreSQL with PostGIS for geospatial queries).

---

## Out of Scope
- No rule-based or hard-coded scoring logic.
- No manual or admin-only match curation.
- No offline/local database.

---

## Problem Statement

Every year, industrial facilities — factories, data centers, refineries — release enormous quantities of heat directly into the atmosphere as waste, a resource that is generated, used once, and discarded. Meanwhile, neighboring facilities like greenhouses, cold-storage units, and district heating networks are actively paying to generate the same thermal energy from scratch. The problem is not scarcity of heat — it is that no infrastructure exists to treat waste heat as a transferable utility. Heat dissipates invisibly, contracts never get written, and billions of dollars of recoverable thermal energy evaporate daily simply because no one has built the pipe between the people burning it off and the people who need it.

HeatREco aims to solve this by using AI to connect those with surplus heat to those who need it, making waste heat a valuable, shared resource.
